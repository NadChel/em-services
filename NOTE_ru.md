# Записка для ревьювера

1. Вся документация и README для единообразия написаны на английском языке.
2. Операции с пользователями отделены в отдельный сервис, так как представляют отдельный блок ответственностей. То же самое с сервисом авторизации, выдающим JWT (access tokens).
3. Так как сервисов теперь несколько, во избежание дублирования скриптов Liquibase, миграция (создание таблиц, типов, добавление ролей `USER` и `ADMIN`) вынесена в отдельный сервис, выполняемый на старте.
4. Статусы карт не включают статус `EXPIRED`. Такое решение было принято для того, чтобы исключить рассогласование статуса и срока действия карты.
5. Дерево директорий немного изменено. В частности, все data-классы находятся в директории `data` (сущности, DTO, `@ConfigurationProperties`).
6. `docs` отсутствует. Эндпоинты описаны в самих контроллерах через аннотации к методам. Это уменьшает дистанцию между кодом и его описанием, снижая риск рассогласованности одного с другим.
7. Git-репозитории пересозданы. Содержат только собственные коммиты, относящиеся к выполнению задания.
8. Префикс "em" означает Effective Mobile – название компании, к вакансии которой относится данное тестовое задание. Так я сгруппировал сервисы, относящиеся к этому заданию. В `docker-compose` префикс опущен.
